Part A · The Admin PanelThe back office. Where the event is built before, and closed after.
Draft · build the event
CREATE

Event Creation Specified

action: create the event with a required name and date. Begins it in draft.
↓
BRAND

Host Identity and Branding Specified

action: type the required host name, optionally upload a logo. This is what every attendee screen wears.
↓
SETUP

Event Setup and Active Event

action: select the active event and open its settings. Scopes everything that follows.
↓
PROMPTS

Prompt and Pairing Settings

action: load the questions for the Drop and tune how pairing behaves. The queue panel.
↓
WRITE

Question Creator

action: write custom questions for the Drop or the survey. Feeds the question bank.
↓
CHIPS

Chip Queue and Moderation

action: review and approve the chip vocabulary so the shared language stays clean.
Ready before doors
The Check-In Panel stands ready for manual sign-ins, and the Room Panel will fill as people arrive. Both wait in the Admin for the moment the event goes live.
The seam between the two tools

Go Live

Event Lifecycle Control flips the event from draft to live. This single action activates the entry QR for attendees and unlocks the acts in the Command Center. Specified in the Event Layer.

Part B · The Command CenterThe live cockpit. One screen the host drives the whole night from.
Always on, the whole night
Four things stay live across every view: the Live Stat Bar (scanned in, responses, pairs, nudges, surveys), the Activity Feed (a running ticker), the Connection Status Badge (the realtime link is healthy), and Lifecycle State Awareness, which confirms the event is live and is the gate that lets the acts fire at all.
Live · run the night
WATCH

Room View (Host)

the default view. The host watches the room fill and reads who is present. The home base of the cockpit.
↓
ACT 1

Sync Control · drives The Drop

action: set a prompt and fire it to every phone at once. Watch the timer and responses land.
↓
ACT 2

Chance Control · drives The Chance

action: pair the room, re-spin if needed. The pairs grid fills as people are matched.
↓
ACT 3

Nudge Control · drives The Nudge

action: send private matches, one to one. Each goes only to its recipient, never to a shared view.
Read and steer, anytime during live
The host can drop into the Intelligence View to read the room's offers, needs, and gaps, scan the Open Mic Feed of live responses, open the Survey Monitor as responses arrive, push a message through Broadcast and Announce, or step back to the Admin Check-In Panel to add someone by hand. Toasts confirm each action across both tools.
The seam back to the Admin

Close the Event

Event Lifecycle Control flips live to closed. The acts can no longer fire, the data is final, and the host returns to the Admin to review and pull it out.
