# SODA: Host Flow Map

*Running an event, Admin to Command Center. A product of Equalpoint, Inc. Current as of the Event Layer update.*

The host journey across both tools, held together by the event lifecycle. The Admin is the back office where an event is built and closed. The Command Center is the live cockpit where the night is run. The host moves between them as the event moves through its states. These are cockpits, not linear screen journeys like the attendee flow, so this maps the order a host actually works in.

## The lifecycle is the spine

- **Draft:** built in the Admin, the QR is not yet live.
- **Live:** run from the Command Center.
- **Closed:** reviewed and exported in the Admin.

## Part A: The Admin Panel

*The back office. Where the event is built before, and closed after.*

### Draft, build the event

1. **Event Creation.** Create the event with a required name and date. Begins it in draft.
2. **Host Identity and Branding.** Type the required host name, optionally upload a logo. This is what every attendee screen wears.
3. **Event Setup and Active Event.** Select the active event and open its settings. Scopes everything that follows.
4. **Prompt and Pairing Settings.** Load the questions for the Drop and tune how pairing behaves. The queue panel.
5. **Question Creator.** Write custom questions for the Drop or the survey. Feeds the question bank.
6. **Chip Queue and Moderation.** Review and approve the chip vocabulary so the shared language stays clean.

Ready before doors: the Check-In Panel stands ready for manual sign-ins, and the Room Panel will fill as people arrive. Both wait in the Admin for go-live.

## The seam: Go Live

Event Lifecycle Control flips the event from draft to live. This single action activates the entry QR for attendees and unlocks the acts in the Command Center.

## Part B: The Command Center

*The live cockpit. One screen the host drives the whole night from.*

Always on, the whole night: four things stay live across every view.

- **Live Stat Bar:** scanned in, responses, pairs, nudges, surveys.
- **Activity Feed:** a running ticker of everything happening.
- **Connection Status Badge:** the realtime link is healthy.
- **Lifecycle State Awareness:** confirms the event is live, and is the gate that lets the acts fire at all.

### Live, run the night

1. **Room View (Host).** The default view. Watch the room fill and read who is present. The home base of the cockpit.
2. **Sync Control, drives The Drop (Act 1).** Set a prompt and fire it to every phone at once. Watch the timer and responses land.
3. **Chance Control, drives The Chance (Act 2).** Pair the room, re-spin if needed. The pairs grid fills as people match.
4. **Nudge Control, drives The Nudge (Act 3).** Send private matches, one to one. Each goes only to its recipient, never to a shared view.

Read and steer, anytime during live: the Intelligence View to read offers, needs, and gaps; the Open Mic Feed to scan live responses; the Survey Monitor as responses arrive; Broadcast and Announce to message the room; the Admin Check-In Panel to add someone by hand. Toasts confirm each action across both tools.

## The seam back: Close the Event

Event Lifecycle Control flips live to closed. The acts can no longer fire, the data is final, and the host returns to the Admin to review and export.

## Part C: Back in the Admin

*Closed. Review the record and pull the data out.*

1. **Matches Panel and Survey Panel.** Review every match the engine surfaced and the full survey data.
2. **Export Panel.** Pull the data out: attendees, connections, survey. Where Futureland's ownership of its own event data is exercised.
3. **Recap goes out.** The recap email fires automatically to attendees. The event is now a closed record.

## The two tools, in short

- **Admin** builds the event in draft and closes it after: event creation, host branding, prompts, chips, check-in, matches, survey, export.
- **Command Center** runs the live night: stat bar, room view, the three act controls, intelligence, survey monitor, broadcast, activity feed.
- **The lifecycle connects them.** Go live moves the host from Admin into the Command Center. Closing moves them back to the Admin to export.
