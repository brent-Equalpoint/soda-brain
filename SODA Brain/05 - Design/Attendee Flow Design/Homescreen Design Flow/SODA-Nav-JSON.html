Each of the four nav tabs is its own JSON object. Switch tabs to see the contract for Rooms, Card, People, and You, and the screen each one renders. Every screen is a projection of server data, the client only paints what it receives.

Four tabs, one pattern. Each screen declares a screen name, a header, and a typed body: Rooms sends a live_room plus a past_rooms array, Card sends the card object, People sends filters plus a people array, You sends profile, stats, settings, and the equalpoint_handoff. The warmth values stay enums so the client owns the styling. Same discipline as the recap: the server decides, the screen renders.
