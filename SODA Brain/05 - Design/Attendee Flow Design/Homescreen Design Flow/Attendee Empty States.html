First-time user, the empty states

What every screen looks like for someone who just signed up and has done nothing yet. The important nuance: a screen can be live and empty at the same time. The hero runs on event_state, the body runs on user_state. A new user during a live event gets a live hero over an empty body, and that is correct.

The rule that makes this clean: an empty state is not a separate screen. It is the same contract carrying state: "empty" and empty arrays. The server decides emptiness, the client swaps the body for an invitation. One renderer, two outcomes. Build the screen once, it handles full and empty.
