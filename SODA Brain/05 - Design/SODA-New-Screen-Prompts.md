# SODA: Paste-Ready Prompts for the New Screens

*Every Claude Design prompt for the screens added this session, in one place. Copy a block, paste it into Claude Design, and it builds against the locked SODA foundation: dark mode, near-black #111111 canvas, Soda green #3BD75C for interactive elements only, condensed uppercase display type, an 8px grid, and one dominant element per screen.*

---

## Access & Sessions

### 1. The code-not-arriving help state

```
Design a mobile screen, dark mode, background near-black #111111. This is the SODA sign-in code screen
in its help state, when the code has not arrived. One job: get the person unstuck.
Top: a slim line, the email the code was sent to in muted grey monospace, with a small "Edit" link in
Soda green #3BD75C beside it.
Center: the six-digit code input, empty, with a calm line of off-white #F5F5F5 text below, "It can take
a moment. Check your spam folder too."
Below: a "Resend code" button that shows a countdown like "Resend in 0:20" in muted grey while waiting,
then becomes an active green text link.
At the bottom, a quiet line in muted grey, "At the event? Ask the host to check you in."
Calm, never alarming. Green only on the links. 8px grid, 16px margins.
```

### 2. The welcome-back re-authentication

```
Design a mobile screen, dark mode, background near-black #111111. This is the SODA welcome-back screen
for a returning person whose session expired. One job: confirm it is them and restore their home.
Top center: the host identity if an event is live, otherwise the quiet SODA mark.
Center, dominant: a warm headline in bold condensed uppercase display type, white, "WELCOME BACK, MAYA".
Below it one line of off-white #F5F5F5 body text, "Confirm it is you to pick up where you left off."
A single six-digit code input beneath, with a note that a code was sent to their email.
Bottom: a full-width Soda green #3BD75C button, "CONTINUE". A quiet "Use a different email" link below.
Feels like a greeting, not a checkpoint. Green only on the button. 8px grid, 16px margins.
```

## Event Mode

### 4. Event Mode control (host setup toggle)

```
Design a section of a host setup screen, dark mode, background near-black #111111. This is the Event
Mode control in SODA event creation. One job: let the host choose Full or Simple.
A small label in muted grey monospace, "EVENT MODE". Below it a segmented two-option toggle in a
dark #1D1F1D track: "Full" and "Simple", each with a tiny subtitle, "The whole experience" and
"Entry and survey only". The selected option fills Soda green #3BD75C with near-black text.
Below the toggle, one calm line of off-white #F5F5F5 text that updates with the choice, for Simple:
"Guests scan in, build a quick profile, and scan again for the survey. The live acts stay off."
A quiet muted-grey link, "Adjust individual acts", hinting the dial underneath.
8px grid, 16px margins. Green only on the selected option. Calm and clear.
```

## How to use these

Paste one block at a time. Each is self-contained and names its own colors, type, and layout, so you do not need to prepend the foundation. If you want a set to share a look, run them in one session so the styling carries. The purple accent (#A47BFF) always means a nudge, the amber (#F59E0B) is only for timers, and green stays reserved for what is interactive or confirming.
