# Test Suite

## Structure

- `tests/unit/frontend/`   — Component and hook tests (Aniya Fronte)
- `tests/unit/backend/`    — API route and service tests (Kennis Beck)
- `tests/integration/`     — Cross-layer integration tests
- `tests/e2e/`             — End-to-end tests (Playwright or Cypress)
- `tests/e2e/fixtures/`    — Shared test data and mocks

## Running Tests

See `docs/STACK.md` for test commands per layer.
Ghost uses this directory structure as the source of truth during verification passes.
