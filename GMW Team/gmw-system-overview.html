00 — Plain English

02 — The Five Phases

Every piece of work moves through all five phases — no skipping, no shortcuts. Each phase has one owner, one output, and one gate that must clear before the next phase begins. The phases are what prevent the team from producing surprises.

01

Name the problem

What's the actual problem? Written clearly, in plain language, in four sentences or fewer. If it's really two problems, it gets split into two separate tickets before anything else happens. Vague problems produce vague software.

The
Non-
Negotiables11

These aren't guidelines. They're locked decisions. Changing any one of them requires a formal spec change and Alysha's approval. No agent can deviate from them — not even the Orchestrator. They exist because some mistakes are too expensive to make twice.

R — 01
Design Before Code

Aniya designs. Alysha approves. Kennis builds from what was approved. The code serves the experience — never the reverse. This is not a preference. It is the sequence.

R — 02
Two Gates to Close

Every ticket needs two separate sign-offs: Ghost says the work is solid, Scribe says the documentation is complete. One alone is not enough to close.

R — 03
No Secrets in Code

API keys, passwords, and tokens never appear in any file. They live in a secrets manager. Committing a secret is an automatic stop — full stop, no exceptions.

R — 04
One Agent at a Time

Only one agent is working at any given moment. Everything routes through the Orchestrator. No agent speaks directly to another. This prevents conflicts and lost work.

R — 05
Security Reviews Auth

Any work touching login, codes, or access control gets a mandatory Cipher security review before Ghost can approve it. No exceptions on auth tickets. Ever.

R — 06
History Is Permanent

Event data is never deleted or rewritten. Only new records get added. This is what makes the data trustworthy for research and makes recovery possible when something goes wrong.

R — 07
Stop and Surface

Any agent that hits ambiguity, a security risk, or a major fork in the road stops immediately, documents exactly what happened, and waits. No guessing forward. No improvising.

R — 08
No Direct to Production

Nothing merges to main or production directly. Every change lives on its own branch and passes Ghost before a merge happens. Without exception.

R — 09
Warmth Words Are Fixed

The relationship warmth system uses exactly two phrases: "in rhythm" when active, "it's been a while" when cooling. No variations. Echo enforces this across every screen that ships.

R — 10
Verify Before the Code

For ops and host logins: the server checks if an email is on the authorized list before generating a code. An unauthorized email gets a neutral response that reveals nothing about why.

R — 11
Categories Are Locked

Profile chips have exactly five category types. Adding a sixth requires a formal spec change — not a code change. This keeps the entire data model consistent across the platform.
